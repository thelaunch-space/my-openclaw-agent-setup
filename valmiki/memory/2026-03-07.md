### Angle 2: "The Non-Technical Vetting Framework"

**insightText:** Non-technical founders struggle to evaluate developers because they cannot read code. The 3-step vetting framework that works without technical knowledge: (1) see working products they built, not portfolios; (2) pay for a 2-week test project before committing; (3) call references and ask "would you hire them again?" Portfolios can be faked. Working software and reference calls cannot.

**rationale:** A 48-year-old legal or healthcare founder needs to hire a developer for case management or client systems but has zero technical background. They do not know how to separate good developers from bad ones. Every hiring guide assumes code literacy. This gives them a concrete 3-step framework that works without reading code - see live products, run a paid test, call references. Immediately actionable for founders hiring their first developer right now.
